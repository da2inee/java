# 다형성 (Polymorphism)

## 다형성이란?
**다형성**은 같은 타입(부모 클래스/인터페이스)으로 여러 구현 객체를 다르게 동작시키는 특성이다.

즉, "같은 호출"인데 "실제 객체 타입"에 따라 결과가 달라진다.

## 왜 필요한가?
- 구현체 교체가 쉬워져 확장성이 좋아진다.
- 조건문(if-else) 분기를 줄이고 역할 중심 설계를 할 수 있다.
- 테스트에서 Mock/Fake 객체로 대체하기 쉽다.

## 예시 코드
```java
interface Payment {
    void pay(int amount);
}

class CardPayment implements Payment {
    @Override
    public void pay(int amount) {
        System.out.println("카드 결제: " + amount);
    }
}

class KakaoPayPayment implements Payment {
    @Override
    public void pay(int amount) {
        System.out.println("카카오페이 결제: " + amount);
    }
}

public class OrderService {
    private final Payment payment;

    public OrderService(Payment payment) {
        this.payment = payment;
    }

    public void checkout(int amount) {
        payment.pay(amount);
    }
}
```

## 실무에서 중요한 포인트
- 다형성은 보통 인터페이스 + 구현체 조합으로 사용한다.
- 구현체가 늘어도 호출 코드는 바꾸지 않는 구조를 목표로 한다.
- 스프링의 의존성 주입(DI)과 함께 쓰면 효과가 더 크다.

## 한 줄 정리
다형성은 "공통 타입으로 추상화하고, 실제 동작은 구현체가 책임지게 하는 것"이다.
