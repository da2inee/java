# 생성자 (Constructor)

객체가 **처음 만들어질 때** 한 번 실행되는 특별한 메서드입니다. "객체를 만들면서 뭘 꼭 넣어두고 싶을 때" 쓰는 거라고 보면 됩니다.

---

## 생성자가 뭐야?

- **이름**이 **클래스 이름과 똑같고**, **리턴 타입이 없음** (void도 안 씀).
- **`new 클래스이름()`** 할 때 **자동으로 한 번** 호출됨.
- 보통 **객체의 필드(변수) 초기값**을 넣을 때 씀.

```java
public class Person {
    String name;
    int age;

    // 생성자: 클래스 이름과 같고, 리턴 타입 없음
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// 사용
Person p = new Person("김자바", 25);  // 이 순간 Person(String, int) 생성자가 실행됨
```

---

## 왜 쓰나?

- 객체를 만들자마자 **이름·나이 같은 값**을 넣어 두고 싶을 때.
- 생성자 없이 하려면 `p.name = "김자바"; p.age = 25;` 처럼 매번 따로 넣어야 함.
- 생성자에 넣어 두면 **`new Person("김자바", 25)` 한 줄**로 "만들면서 바로 초기화" 가능.

→ **객체가 막 생성될 때 꼭 필요한 값들을 한 번에 받아서 세팅**하는 용도.

---

## 기본 생성자 (Default Constructor)

- **생성자를 하나도 안 쓰면** 컴파일러가 **인자 없는 생성자**를 알아서 만들어 줌.
- 이걸 **기본 생성자**라고 부름: `클래스이름() { }`

```java
public class Box {
    int size;
    // 생성자 없음 → 컴파일러가 Box() { } 자동 추가
}

Box b = new Box();  // OK. 기본 생성자 호출
```

- **생성자를 하나라도 직접 쓰면** 기본 생성자는 **안 만들어짐**.

```java
public class Person {
    String name;

    Person(String name) {
        this.name = name;
    }
}

Person p = new Person("김");  // OK
Person q = new Person();     // 컴파일 에러! 인자 없는 생성자 없음
```

→ 생성자를 오버로딩해서 쓰면, **필요하면 `Person()` 같은 인자 없는 생성자도 직접 하나 써줘야** `new Person()` 이 가능함.

## this / this()

- **this**  
  "이 객체 자신"을 가리킴.  
  필드 이름이 매개변수랑 같을 때 구분하려고 씀.

```java
Person(String name, int age) {
    this.name = name;   // this.name = 필드, name = 매개변수
    this.age = age;
}
```

- **this()**  
  **같은 클래스의 다른 생성자**를 호출할 때 씀.  
  반드시 **생성자 안에서, 첫 줄**에만 쓸 수 있음.

```java
public class Person {
    String name;
    int age;

    Person(String name) {
        this(name, 0);  // 아래 생성자 호출 → name 넣고 age는 0
    }

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

---

## 생성자 오버로딩

- **같은 이름(클래스 이름)**, **매개변수 개수/타입만 다르게** 여러 개 두는 것.
- `new Person("김")` / `new Person("김", 25)` 처럼 **다른 방식으로 객체 생성** 가능.

```java
Person() { ... }
Person(String name) { ... }
Person(String name, int age) { ... }
```

---

## 정리 (면접용)

| 질문 | 답변 |
|------|------|
| 생성자가 뭐야? | 객체가 **처음 생성될 때** 한 번 호출되는, **클래스 이름과 같은** 특별한 메서드. 리턴 타입 없음. |
| 왜 쓰나? | 객체 **필드 초기화** (만들면서 값을 넣어 두기 위해). |
| 기본 생성자? | 생성자를 **하나도 안 쓰면** 컴파일러가 `클래스이름() { }` 를 자동 추가. 하나라도 쓰면 안 만들어짐. |
| this / this()? | **this** = 이 객체 자신. **this()** = 같은 클래스의 다른 생성자 호출 (생성자 첫 줄에서만). |

## 예제 한 번에

```java
public class Coffee {
    String name;
    int price;

    Coffee() {
        this("기본", 0);  // 아래 생성자 호출
    }

    Coffee(String name) {
        this(name, 4500);
    }

    Coffee(String name, int price) {
        this.name = name;
        this.price = price;
    }

    public static void main(String[] args) {
        Coffee c1 = new Coffee();           // name="기본", price=0
        Coffee c2 = new Coffee("라떼");      // name="라떼", price=4500
        Coffee c3 = new Coffee("아메", 4000); // name="아메", price=4000
    }
}
```

이 정도만 알아두면 생성자 개념은 충분합니다.
