# 캡슐화 (Encapsulation)

## 캡슐화란?
**캡슐화**는 객체의 데이터를 외부에서 직접 변경하지 못하게 숨기고,  
정해진 메소드로만 접근/변경하게 만드는 설계 방식이다.

핵심은 "데이터 보호"와 "올바른 사용 강제"다.

## 왜 필요한가?
- 잘못된 값 입력을 막아 데이터 무결성을 지킨다.
- 내부 구현을 숨겨 변경 영향 범위를 줄인다.
- 객체 책임을 명확히 해서 유지보수를 쉽게 만든다.

## 예시 코드
```java
public class Account {
    private int balance; // 외부에서 직접 접근 불가

    public void deposit(int amount) {
        if (amount <= 0) throw new IllegalArgumentException("0보다 커야 함");
        balance += amount;
    }

    public void withdraw(int amount) {
        if (amount <= 0) throw new IllegalArgumentException("0보다 커야 함");
        if (balance < amount) throw new IllegalStateException("잔액 부족");
        balance -= amount;
    }

    public int getBalance() {
        return balance;
    }
}
```

## 실무에서 중요한 포인트
- 필드는 기본적으로 `private`로 선언한다.
- `setXxx()`를 무조건 열지 말고, 의미 있는 행위 메소드로 상태를 바꾼다.
  - 예: `setStatus("DONE")`보다 `complete()`가 의도 전달에 좋다.
- 상태가 바뀌는 메소드 안에서 검증을 강제한다.

## 한 줄 정리
캡슐화는 "데이터를 숨기고, 객체가 스스로 자신의 규칙을 지키게 만드는 것"이다.
