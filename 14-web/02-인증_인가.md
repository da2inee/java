# 인증(Authentication) / 인가(Authorization)

## 인증(Authentication)이란?
**“너 누구야?”**를 확인하는 과정.
- 예: 아이디/비밀번호 로그인, 소셜 로그인, OTP 등
- 결과: “이 요청은 사용자 A가 보낸 게 맞다”를 확인

## 인가(Authorization)란?
**“너 이거 해도 돼?”**를 확인하는 과정.
- 예: 관리자만 접근 가능한 API, 본인 글만 수정 가능
- 결과: “사용자 A가 이 리소스/행위를 할 권한이 있다”를 확인

## 왜 분리해서 생각해야 해?
- 인증은 통과했어도(로그인 했어도) **모든 권한이 생기는 건 아님**
- 인가 규칙은 기능마다 다르고 자주 바뀌어서, 인증과 분리하면 설계가 깔끔해진다.

## 웹에서 자주 보는 예시
- **인증**: 토큰/JSESSIONID가 유효한가?
- **인가**: 역할이 `ADMIN`인가? 또는 리소스의 소유자가 본인인가?

## 실무 팁
- 인가는 “역할 기반(Role)”만으로 끝나지 않는 경우가 많다.
  - 예: `게시글.ownerId == 로그인유저.id` 같은 **리소스 기반 권한**이 필수인 경우가 많음

## 한 줄 정리
인증은 **정체 확인**, 인가는 **권한 확인**이다.
