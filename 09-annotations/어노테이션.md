# 어노테이션 (Annotation)

**어노테이션** = 코드에 붙이는 **메타데이터(표시)**.  
클래스, 메소드, 필드 위에 `@이름` 형태로 붙여서, **컴파일러나 프레임워크(Spring 등)**가 그걸 읽고 동작을 바꿀 수 있게 해요.

---

## 어노테이션이 뭐야?

- **형태**: `@이름` 또는 `@이름(속성=값)`
- **역할**: "이건 이렇게 써라", "이건 이렇게 처리해라" 같은 **정보**를 코드에 달아 두는 것.
- **누가 읽나**: 컴파일러(JVM), 또는 **리플렉션**으로 런타임에 읽는 라이브러리·프레임워크(Spring, JPA 등).

```java
@Override
public String toString() { ... }

@RequestParam("name") String name
```

---

## 자바 기본 어노테이션

| 어노테이션 | 의미 |
|------------|------|
| **@Override** | "이 메소드는 부모 메소드를 덮어쓴다." 컴파일러가 부모에 그 메소드 있는지 검사함. |
| **@Deprecated** | "이건 쓰지 말고 다른 걸 써라." 사용 시 경고 나옴. |
| **@SuppressWarnings("unchecked")** | 특정 경고를 이 구간에서만 무시. |

---

## Spring에서 요청 데이터 받을 때

| 어노테이션 | 어디서 받나 | 예시 |
|------------|-------------|------|
| **@RequestParam** | 쿼리 스트링 (`?key=value`) | `?name=김&age=25` |
| **@RequestBody** | 요청 본문(JSON 등) → 객체로 바인딩 | `{"name":"김"}` |
| **@PathVariable** | URL 경로 안 값 | `/users/{id}` → `id` |
| **@RequestHeader** | HTTP 헤더 | `Authorization`, `Content-Type` |
| **@CookieValue** | 쿠키 | `JSESSIONID` |
| **@ModelAttribute** | 쿼리/폼을 객체 한 번에 | `?name=김` → DTO 필드에 바인딩 |

---

## Spring에서 자주 쓰는 것

| 어노테이션 | 붙이는 곳 | 의미 |
|------------|-----------|------|
| **@SpringBootApplication** | main 있는 클래스 | "이게 Spring Boot 앱 시작점이다." |
| **@Component** | 클래스 | "이 클래스는 빈으로 등록해라." |
| **@Service** | 클래스 | 비즈니스 로직용 빈 (실제로는 @Component와 동작 비슷) |
| **@Controller** | 클래스 | 웹 요청 처리용 컨트롤러 |
| **@RestController** | 클래스 | @Controller + 응답을 JSON 등으로 (메소드마다 @ResponseBody 생략) |
| **@Bean** | 메소드 | "이 메소드가 반환하는 객체를 빈으로 등록해라." |
| **@Autowired** | 필드/생성자/세터 | "이 타입의 빈을 주입해라." (의존성 주입) |
| **@Value** | 필드 | 설정값(properties) 주입. 예: `@Value("${server.port}")` |
| **@Profile** | 클래스/빈 | "이 프로필일 때만 이 빈을 써라." (dev, prd 등) |
| **@RequestMapping** | 클래스/메소드 | URL 매핑. **@GetMapping**, **@PostMapping** 등으로 구체화. |

---

## 정리

- **어노테이션** = `@이름` 형태의 **메타데이터**. 컴파일러·프레임워크가 읽고 동작을 정함.
- **자바 기본**: @Override, @Deprecated, @SuppressWarnings
- **Spring 요청**: @RequestParam, @RequestBody, @PathVariable, @RequestHeader, @CookieValue, @ModelAttribute
- **Spring 구조**: @Component, @Service, @Controller, @RestController, @Bean, @Autowired, @Value, @Profile, @GetMapping 등

실제로는 **어디에 붙이냐**(클래스/메소드/파라미터)에 따라 의미가 정해지므로, 위 표를 보면서 코드에서 `@` 가 붙은 부분을 하나씩 대응해 보면 됨.
