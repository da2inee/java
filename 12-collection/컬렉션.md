# 컬렉션 (Collection)

**여러 개의 데이터**를 담아두고 다루기 위한 **자바 표준 자료구조**예요.  
배열은 크기가 고정인데, 컬렉션은 **동적으로 추가·삭제**할 수 있어요.

---

## 1. List – 순서 O, 중복 O

**ArrayList**가 가장 많이 쓰여요. 인덱스로 접근하고, 순서가 보장돼요.

### 기본 사용

```java
import java.util.ArrayList;
import java.util.List;

List<String> names = new ArrayList<>();
names.add("김철수");
names.add("이영희");
names.add("김철수");   // 중복 허용

System.out.println(names);           // [김철수, 이영희, 김철수]
System.out.println(names.get(1));    // 이영희 (인덱스 1)
System.out.println(names.size());    // 3
```

### 자주 쓰는 메서드

| 메서드 | 설명 |
|--------|------|
| `add(e)` | 맨 뒤에 추가 |
| `add(i, e)` | 인덱스 i에 삽입 |
| `get(i)` | 인덱스 i 값 조회 |
| `set(i, e)` | 인덱스 i 값 변경 |
| `remove(i)` | 인덱스 i 삭제 (또는 `remove(Object)`로 요소로 삭제) |
| `size()` | 요소 개수 |
| `contains(e)` | 포함 여부 |

### 예시: 반복문으로 출력

```java
for (int i = 0; i < names.size(); i++) {
    System.out.println(names.get(i));
}
// 또는
for (String name : names) {
    System.out.println(name);
}
```

## 2. Set – 순서 보장 X (구현에 따라 다름), 중복 X

**HashSet**이 가장 많이 쓰여요. 같은 값을 여러 번 넣어도 **하나만** 들어가요.

### 기본 사용

```java
import java.util.HashSet;
import java.util.Set;

Set<String> ids = new HashSet<>();
ids.add("a");
ids.add("b");
ids.add("a");   // 중복 → 무시

System.out.println(ids);    // [a, b] (순서는 보장 안 됨)
System.out.println(ids.size());   // 2
```

### 자주 쓰는 메서드

| 메서드 | 설명 |
|--------|------|
| `add(e)` | 추가 (이미 있으면 false 반환) |
| `remove(e)` | 삭제 |
| `contains(e)` | 포함 여부 |
| `size()` | 요소 개수 |

### 예시: 중복 제거

```java
List<String> list = List.of("가", "나", "가", "다");
Set<String> set = new HashSet<>(list);
System.out.println(set);   // [가, 나, 다]
```

## 3. Map – 키(key)와 값(value) 쌍

**HashMap**이 가장 많이 쓰여요. 키로 값을 찾아요. 키는 중복 불가, 값은 중복 가능.

### 기본 사용

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> scores = new HashMap<>();
scores.put("김철수", 90);
scores.put("이영희", 85);
scores.put("김철수", 95);   // 같은 키 → 값 덮어씀

System.out.println(scores.get("김철수"));   // 95
System.out.println(scores.size());         // 2
```

### 자주 쓰는 메서드

| 메서드 | 설명 |
|--------|------|
| `put(key, value)` | 키-값 저장 (같은 키면 값 갱신) |
| `get(key)` | 키로 값 조회 (없으면 null) |
| `getOrDefault(key, default)` | 없으면 기본값 반환 |
| `remove(key)` | 키 삭제 |
| `containsKey(key)` | 키 존재 여부 |
| `keySet()` | 키들만 Set으로 |
| `values()` | 값들만 Collection으로 |

### 예시: 키만 순회

```java
for (String name : scores.keySet()) {
    System.out.println(name + " : " + scores.get(name));
}
```

### 예시: 키-값 쌍 순회

```java
for (Map.Entry<String, Integer> e : scores.entrySet()) {
    System.out.println(e.getKey() + " : " + e.getValue());
}
```

## 4. List vs Set vs Map (한눈에)

| 구분 | List | Set | Map |
|------|------|-----|-----|
| **대표 구현** | ArrayList | HashSet | HashMap |
| **순서** | O (인덱스) | X (HashSet은 무순서) | X (키 기준 저장) |
| **중복** | O | X | 키 중복 X, 값 중복 O |
| **접근** | `get(i)` | `contains(e)` 등 | `get(key)` |

## 5. 제네릭 (Generics)

컬렉션은 **제네릭**으로 요소 타입을 정해요. `<>` 안에 타입을 쓰면 그 타입만 넣을 수 있어요.

```java
List<Integer> numbers = new ArrayList<>();
numbers.add(1);
numbers.add(2);
// numbers.add("hello");   // 컴파일 에러

Map<String, List<Integer>> map = new HashMap<>();   // 키: String, 값: List<Integer>
```

---

## 6. 면접용 정리

| 질문 | 답변 |
|------|------|
| List vs Set? | **List** = 순서 O, 중복 O. **Set** = 중복 X. |
| ArrayList vs LinkedList? | **ArrayList** = 인덱스 접근 빠름. **LinkedList** = 앞뒤 삽입/삭제에 유리. |
| HashMap 키 기준? | **equals**, **hashCode**로 동일 여부 판단. 키는 불변에 가깝게 쓰는 게 안전. |
